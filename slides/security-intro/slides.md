# 什麼是資訊安全？

---

## 資訊安全（Information Security）

保護**資訊**不被未經授權的人存取、竄改或破壞

<br>

換句話說，就是確保：

| 目標                 | 說明   | 日常例子       |
| -------------------- | ------ | -------------- |
| 資訊只有該看的人能看 | 機密性 | 帳號密碼不外洩 |
| 資訊不被惡意修改     | 完整性 | 成績單沒被人改 |
| 需要時隨時可以取得   | 可用性 | 網站沒被打掛   |

---

## 為什麼高中生也需要懂資安？

你每天都在用：

- 社群媒體（Instagram、Discord、Threads）
- 線上遊戲帳號
- 電子郵件、Google 帳號
- 學校系統、各種 App

<br>

> 帳號被盜、個資外洩、詐騙，**任何人**都可能是目標。

# CIA 三元素

資安最核心的三個概念

---

## Confidentiality — 機密性

只有**有權限的人**才能存取資訊

<br>

**現實例子：**

- 你的 IG 私訊只有你跟對方能看
- 學校的成績系統需要帳號才能登入

<br>

**威脅：** 未授權存取、資料外洩、帳號被盜

---

## Integrity — 完整性

資訊在傳輸或儲存過程中**沒有被竄改**

<br>

**現實例子：**

- 下載的軟體跟官網上的一模一樣（MD5/SHA256 校驗）
- 你傳給朋友的訊息沒有被人中途修改

<br>

**威脅：** 中間人攻擊、資料竄改

---

## Availability — 可用性

系統與資訊在**需要的時候**能夠正常使用

<br>

**現實例子：**

- 考試期間學校系統不能掛掉
- 遊戲伺服器 24 小時正常運作

<br>

**威脅：** DDoS 攻擊、系統崩潰


<br>

> 技術再強，人還是最大的漏洞。

# 惡意軟體（Malware）

---

## 什麼是惡意軟體？

**Mal**icious Soft**ware** — 專門設計來破壞、竊取、控制你電腦的程式

<br>

| 種類                  | 行為                   | 傳播方式           |
| --------------------- | ---------------------- | ------------------ |
| 病毒 (Virus)          | 感染其他檔案、自我複製 | 開啟受感染的檔案   |
| 木馬 (Trojan)         | 偽裝成正常程式、留後門 | 下載來路不明的軟體 |
| 間諜軟體 (Spyware)    | 偷偷記錄你的行為       | 捆綁安裝           |
| 勒索軟體 (Ransomware) | 加密你的檔案要贖金     | 惡意郵件附件       |

---

## 勒索軟體有多嚴重？

**2021 年 Colonial Pipeline 事件：**

- 美國最大輸油管公司被勒索軟體攻擊
- 東岸汽油供應中斷 6 天
- 支付了 440 萬美金贖金

<br>

**2023 年台灣某醫院：**

- 病歷系統被加密、無法存取
- 醫療作業被迫回歸紙本

<br>

> 不是只有電影情節，它真的在發生。

---

## 如何防範惡意軟體？

- **只從官方管道下載軟體**，不用破解版
- 定期更新作業系統與防毒軟體
- 不隨意開啟來路不明的郵件附件
- 重要資料定期備份（3-2-1 備份原則）

<br>

**3-2-1 備份原則：**

> 3 份資料 → 2 種媒介 → 1 份放異地（雲端）

# 釣魚攻擊（Phishing）

---

## 釣魚攻擊是什麼？

偽裝成合法來源，騙你提供帳號密碼或點惡意連結

<br>

就像真正的釣魚：

```
攻擊者（漁夫）→ 假網站/假信件（魚餌）→ 你（魚）
```

---

## 釣魚攻擊長什麼樣子？

**案例 1 — 假冒 Google 的信件：**

```
寄件人：security-noreply@gooogle.com   ← 多了一個 o
主旨：您的帳號存在異常登入，請立即驗證

[點我驗證帳號]   ← 連到假的 Google 登入頁
```

<br>

**案例 2 — LINE 假客服：**

```
【LINE客服】您的帳號將於24小時後停用，
請點擊以下連結完成驗證：http://lin3-verify.xyz
```

---

## 怎麼識破釣魚？

**看寄件人網域**

- `@google.com` ✅
- `@gooogle.com` ❌ / `@google.com.phishing.net` ❌

**看網址**

- `https://accounts.google.com` ✅
- `https://accounts-google.verify-now.xyz` ❌

**感覺很緊迫？先暫停**

> 「限時」「馬上」「帳號將停用」— 這是讓你慌亂做錯誤決定的手法

**不確定就直接去官網，不要點信裡的連結**

# 社交工程（Social Engineering）

---

## 不用 hack 電腦，直接 hack 人

社交工程是透過**心理操控**讓人自願交出資訊或執行危險動作

<br>

**攻擊者利用的人性弱點：**

| 弱點 | 手法舉例                         |
| ---- | -------------------------------- |
| 信任 | 假冒同學、老師、客服             |
| 恐懼 | 「你的帳號被入侵了！」           |
| 好奇 | 「你知道你的秘密照流出去了嗎？」 |
| 貪婪 | 「你中獎了，點這裡領獎」         |
| 急迫 | 「限時 1 小時，立刻行動」        |

---

## 社交工程真實案例

**案例：假冒 IT 部門**

```
（電話）
攻擊者：「您好，我是學校 IT 組，
         我們系統升級需要驗證您的帳號，
         請提供您的密碼。」

正確回應：「IT 人員永遠不需要你的密碼」
```

<br>

**記住：**

- 任何人都不應該要求你的密碼
- 不確定就掛電話，直接打官方電話確認
- 在社群媒體不要公開太多個人資訊

# 密碼安全

---

## 什麼是弱密碼？

**全球最常見的密碼（每年都在前十）：**

```
123456
password
123456789
qwerty
abc123
```

<br>

> 攻擊者有一份包含數億筆常見密碼的字典檔（Dictionary Attack），這些密碼在幾秒鐘內就會被破解。

---

## 強密碼的要素

**長度 > 複雜度**

| 密碼                        | 破解難度           |
| --------------------------- | ------------------ |
| `P@ss1!`                    | 幾小時內（太短）   |
| `correcthorsebatterystaple` | 幾百年（長但好記） |
| `Tr0ub4dor&3`               | 幾天（短且難記）   |

<br>

**最佳實踐：**

- 至少 12 個字元以上
- 每個網站用**不同**密碼
- 使用**密碼管理器**（Bitwarden、1Password）

---

## 密碼管理器

你只需要記住**一個主密碼**，其他交給軟體管理

<br>

**推薦工具（免費）：**

- **Bitwarden** — 開源、跨平台、免費
- **KeePassXC** — 本機端儲存，不上雲

<br>

**密碼管理器做什麼：**

- 幫你產生強隨機密碼（如 `kX9#mP2@qLv7`）
- 記住所有網站的帳密
- 自動填入登入表單

# 多因素驗證（MFA）

---

## 只有密碼還不夠

**驗證身分的三種因素：**

| 因素     | 說明       | 例子           |
| -------- | ---------- | -------------- |
| 你知道的 | Knowledge  | 密碼、PIN      |
| 你擁有的 | Possession | 手機、硬體金鑰 |
| 你本身的 | Inherence  | 指紋、臉部辨識 |

<br>

**多因素驗證（MFA）= 同時需要 2 種以上**

---

## 啟用 2FA 的重要性

假設你的密碼在某次資料外洩中被取得：

```
沒有 2FA：
攻擊者有密碼 → 直接登入 ✅ (for attacker)

有 2FA：
攻擊者有密碼 → 還需要你的手機驗證碼 ❌ (for attacker)
```

<br>

**立刻去啟用 2FA 的地方：**

- Google 帳號
- Discord
- Instagram
- GitHub

# 安全上網習慣

---

## 看網址列

**HTTPS vs HTTP**

```
https://www.google.com   ← 加密傳輸，安全
http://www.google.com    ← 明文傳輸，危險（尤其輸入密碼時）
```

<br>

HTTPS 會在瀏覽器顯示一個**鎖頭圖示** 🔒

<br>

> 注意：HTTPS 只代表傳輸加密，不代表網站本身是合法的！釣魚網站也可以有 HTTPS。

---

## 公共 Wi-Fi 的風險

**咖啡廳、捷運、機場的免費 Wi-Fi：**

```
你 ←→ 公共 Wi-Fi ←→ 網際網路
            ↑
     攻擊者可能在這裡監聽（中間人攻擊）
```

<br>

**降低風險的做法：**

- 不要在公共 Wi-Fi 輸入密碼或信用卡號
- 使用 VPN（但注意選擇可信任的 VPN）
- 改用手機熱點處理敏感事項

---

## 軟體更新很重要

**為什麼要更新？**

更新通常包含**安全性修補**（Security Patch）

```
漏洞被發現 → 廠商釋出修補 → 你更新 → 安全
                              ↓ 沒更新
                           攻擊者繼續利用此漏洞
```

<br>

**2017 年 WannaCry 勒索軟體：**

- 利用 Windows 已知漏洞（微軟已發補丁）
- 感染 150 個國家超過 23 萬台電腦
- 原因：很多電腦**沒有更新**

# 隱私保護

---

## 你在網路上留下什麼？

每當你上網，你都在留下**數位足跡**：

- 你瀏覽了哪些網頁
- 你在哪裡、什麼時間登入
- 你的 IP 位址
- 你的裝置資訊

<br>

這些資訊被廣告商、網站、甚至攻擊者所收集。

---

## 個資保護意識

**不要輕易公開的資訊：**

- 真實姓名 + 學校 + 班級
- 電話號碼
- 家庭住址
- 每日行程規律

<br>

**社群媒體設定：**

- 定期檢查哪些人可以看到你的貼文
- 關掉貼文的位置標記
- 不接受陌生人的好友請求

---

## 帳號安全健檢

**定期做以下事情：**

- 檢查 [haveibeenpwned.com](https://haveibeenpwned.com) — 看你的信箱有沒有在資料外洩清單中
- 確認哪些第三方 App 有授權你的 Google/FB 帳號
- 移除你不用的 App 授權
- 定期登出不常用的裝置

# CTF 是什麼？

---

## Capture The Flag

資安競賽！找到隱藏的**旗子（Flag）**

```
Flag 格式通常長這樣：
FLAG{this_is_a_secret_flag}
```

<br>

**解題類型：**

| 類型      | 說明                         |
| --------- | ---------------------------- |
| Web       | 找網站漏洞                   |
| Pwn       | 利用程式漏洞                 |
| Rev       | 逆向工程（分析程式在做什麼） |
| Crypto    | 密碼學解謎                   |
| Forensics | 數位鑑識、隱寫術             |
| Misc      | 雜項                         |

---

## CTF 入門推薦

**新手友善平台：**

- **picoCTF** — 卡內基梅隆大學辦的，題目有難度分級
- **Hack The Box** — 更進階，有引導式學習
- **TryHackMe** — 互動式學習，很適合初學者

<br>

**台灣的比賽：**

- AIS3（Advanced Information Security Summer School）
- 各大學 CTF 校隊舉辦的公開賽

# 如何入門資安？

---

## 學習路線圖

```
基礎打好（必備）
├── 作業系統：Linux 基本指令
├── 網路基礎：TCP/IP、HTTP、DNS
└── 程式基礎：Python（最推薦）

選擇方向
├── 攻擊面（Offensive）→ 滲透測試、CTF
│   └── 工具：Kali Linux、Burp Suite、Nmap
└── 防禦面（Defensive）→ 藍隊、SOC
    └── 工具：Wireshark、SIEM

實戰練習
└── CTF → Bug Bounty → 實際專案
```

---

## 推薦學習資源

**免費線上課程：**

- **TryHackMe** — 互動式學習房間
- **Cybrary** — 資安課程平台
- **YouTube：John Hammond** — CTF 解題影片

**書籍：**

- 《駭客大曝光》（Hacking Exposed）
- 《The Web Application Hacker's Handbook》

**社群：**

- HITCON（台灣駭客年會）社群
- 各大學資安社

## 今天的重點回顧

- **CIA** — 機密性、完整性、可用性，資安三大目標
- **惡意軟體** — 只從官方來源下載，定期備份
- **釣魚攻擊** — 看清網域、不要因為「緊急」就點
- **社交工程** — 沒人需要你的密碼
- **密碼安全** — 長密碼 + 每站不同 + 密碼管理器
- **MFA** — 開啟兩步驟驗證，帳號多一層保護
- **好習慣** — HTTPS、不亂連 Wi-Fi、保持更新
- **CTF** — 學資安最好玩的方式之一

<br>

> 資安不是目的地，是一種持續的習慣。
